Troubleshooting

If you experience "Authentication required" errors on live environments despite sending a valid token, the Authorization header might be stripped by Apache.

Fix: Add this to your .htaccess: 

SetEnvIf Authorization "(.*)" HTTP_AUTHORIZATION=$1
Create New Opaque Token
For SG AI protected endpoints, set a valid FE user UID. Otherwise the token authenticates, but SG AI account-bound endpoints reject it.
